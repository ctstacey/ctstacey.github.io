Headings

Prefix your heading by 1-6 x # and a space

markdown html 
# Heading1
 
<h1>Heading1</h1>
 
## Heading2
 
<h2>Heading2</h2>
... 
###### Heading6
 
<h6>Heading6</h6>
Only for <h1> and <h2>, can also use ...
 
Heading1
========      // only need one =
Heading2
--------      // only need one -

which produces

<h1>Heading1</h1>
<h2>Heading2</h2>

bold and emphasis

markdown html
_emphasised_
*emphasised*		 <em>emphasised</em>
__bold__
**bold**		 <b>bold</b>

Code

Inline Code

Surround code with backticks (`...`) and it will be enclosed by <code> </code>

markdown html
my `inline code` here my <code>inline code</code> here

Blocks of Code

Indent block by 4 spaces and code will be enclosed by <pre><code> </code></pre>

Example
 

....def function(){
      pass
    }

produces

<pre class="language-text">def function(){
  pass
}
</pre>

which looks like

def function(){
  pass
}

Atom's Code Fences

Atom's Markdown Preview (and Markdown Preview Enhanced) extend this idea by adding code fences with language specified to allow code highlighting (eg ```javascript ...```).

(language defaults to text which looks like regular markdown code block)

Example
 
```javascript
function myFunction() {
  console.log("hello");
}
```

produces (or similar)

<pre class="language-javascript"><span class="token keyword">function</span> <span class="token function">myFunction</span><span class="token punctuation">(</span><span class="token punctuation">)</span> <span class="token punctuation">{</span>
  console<span class="token punctuation">.</span><span class="token function">log</span><span class="token punctuation">(</span><span class="token string">"hello"</span><span class="token punctuation">)</span><span class="token punctuation">;</span>
<span class="token punctuation">}</span>
</pre>

which looks like (or similar)

function myFunction() {
  console.log("hello");
}

Tables

  • Each table row must be fully specified on a single line of the markdown document
  • Put | between all table cells
  • Second row of table must contain the pattern -|-
    (-|- for two columns, -|-|- for three columns, etc)
  • First line without | ends the table
Example
 
header cell 1         | header cell 2
----------------------|----------------------
single `line` content | multi-line<br>content

Note:

  • |'s do not need to line up
  • only need -|- for two column table
  • |'s do not need surrounding spaces

produces

<table>
  <thead>
    <tr>
      <th>header cell 1</th>
      <th>header cell 2</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td>single <code>line</code> content</td>
      <td>multi-line<br>content</td>
    </tr>
  </tbody>
</table>

which looks like

header cell 1 header cell 2
single line content multi-line
content

Single-Line Limitation

The entire table cell content must be on one line of the markdown document
But, you can use <br> to pack multiple lines onto one line.

Unfortunately, anything requiring multi-line syntax (like code blocks) cannot be included in a markdown table.
Instead, use a regular html table in the markdown document.

HTML in markdown documents

Markdown files can include regular HTML.

  • block level elements (<div> <table> <pre> <p> etc)

    • leave a blank line above and below
    • first and last elements should not be indented
    • markdown syntax is not parsed inside the block

  • span level elements (<span> <cite> <del> etc)

    • markdown syntax is parsed
Example (notice: the markdown styling 'overides' the <span> element's local styling)
 
Normal <span style="color:blue;">but **awesome** markdown</span> paragraph
 
<table>
  <tr>
    <td>Block level html</td>
  </tr>
</table>
 
Another normal markdown paragraph

produces

<p>Normal <span style="color:blue;">but <strong>awesome</strong> markdown</span> paragraph</p>
<table>
  <tbody>
    <tr>
      <td>Block level html</td>
    </tr>
  </tbody>
</table>
<p>Another normal markdown paragraph</p>

which looks like

Normal but awesome markdown paragraph

Block level html

Another normal markdown paragraph

Escaped Characters

In HTML, < and & must always be escaped (&lt; and &amp;), otherwise they mess up the parsing of the file.

In markdown, these are automatically escaped for you,
except when they are part of an escape sequence like &amp;, &copy; or &#8203; etc

Escaping markdown syntax

To escape markdown syntax, simply prefix with \

Example
 
> This is a block quote
 
\> This is not

produces

<blockquote>
<p>This is a block quote</p>
</blockquote>
<p>> This is not</p>

which looks like

This is a block quote

> This is not

Paragraphs and linebreaks

Write text on consecutive lines surrounded by blank lines and it will be enclosed by <p> </p>

In vanilla markdown, <br>s are only added when you end a line with two or more spaces.
In Atom's markdown, <br>s are added at the end of every line.

Block Quotes

Consecutive lines prefixed with > will be enclosed by <blockquote> </blockquote>

All other markup rules can be applied inside blockquotes exactly as they would elsewhere.

Example
 

> I showed my boss the `awesome` function I wrote
> ```javascript
> function awesome() {
>   releaseMyCats();
> }
> ```
> and my boss replied
> > You'd better come see
> > me after your done there
>
> the following empty line ends the quote block

produces

<blockquote>
<p>I showed my boss the <code>awesome</code> function I wrote</p>
<pre data-role="codeBlock" data-info="javascript" class="language-javascript"><span class="token keyword">function</span> <span class="token function">awesome</span><span class="token punctuation">(</span><span class="token punctuation">)</span> <span class="token punctuation">{</span>
  <span class="token function">releaseMyCats</span><span class="token punctuation">(</span><span class="token punctuation">)</span><span class="token punctuation">;</span>
<span class="token punctuation">}</span>
</pre><p>and my boss replied</p>
<blockquote>
<p>You'd better come see<br>
me after your done there</p>
</blockquote>
<p>the following empty line ends the quote block</p>
</blockquote>

which looks like

I showed my boss the awesome function I wrote

function awesome() {
  releaseMyCats();
}

and my boss replied

You'd better come see
me after your done there

the following empty line ends the quote block

Gotchas

blank lines within HTML content (within your markdown doc)

  • problem: css defined in <style> element is not applying
    solution: don't leave any blank lines within the <style> element.
    <p></p> will be added invisibly, stopping the styling from applying.
    eg. here, the blank line will stop .two being applied
    <html><head>
<style>
  .one {
    width: 50%;
  }

  .two {
    width: 70%;
  }
</style>
</head></body></html>

I think the problem occurs any time you include HTML inside a markdown document
(at least, it seems to when using Atom's Markdown Preview Enhanced package).
